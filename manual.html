1このツールでできること

土木・建築現場の朝礼ネタを 155 件以上収録した、無料の Web アプリです。インストール不要、ブラウザを開くだけ。スマホ・タブレット・PC、どの端末からでも同じように使えます。

主な機能

  • 📅 今日のおすすめが時期・季節・曜日に合わせて自動表示
  • 🔍 作業内容／天候／対象者／雰囲気の4軸タグでしぼり込み
  • 🔊 音声読み上げで朝礼台本がそのまま読める
  • ⭐ お気に入り保存／使った記録／👍ウケた／📝役に立った を1タップ
  • 💬 公式LINE 毎朝7:00 自動配信＋キーワード送信で関連ネタ返信
  • 📱 看板用 QRコード 自動生成、誰でもスマホで開ける
  • 💾 マイデータバックアップで他端末にデータ移動可能
  • ✏️ 管理者なら ネタ追加・編集・削除もブラウザだけで完結

💴 完全無料

ログイン・アカウント作成・課金、全て不要です。URLにアクセスするだけ。

3🏠 ホーム ── 今日のおすすめを開く

アプリを開くと一番最初に表示される画面。朝、現場に着いたらまずここを開いてください。

表示の構成（上から）

  1. 今日の自動レコメンド ─ 月／曜日／月末などから自動選出した「一番おすすめのネタ」。この台本をひらくで本文が開きます。
  2. 今日のメイン作業は？ ─ クレーン・コンクリ・運転・高所など6つのカテゴリボタン。タップすると「さがす」タブにジャンプして自動でしぼり込まれます。
  3. 今の時期のおすすめ ─ 今月該当のネタが下に続きます。下にスワイプして眺めてください。

💡 一番ラクな使い方

「今日の自動レコメンド」のこの台本をひらくを押すだけで、朝礼で話す内容が用意できます。読み上げ機能（🔊音声）と組み合わせれば、考える時間ゼロ。

4🔍 さがす ── タグでしぼり込む

4種類のタグを掛け合わせて、目当てのネタを絞り込みます。

使えるタグの種類

タグ群例
作業内容クレーン・重機 / コンクリート打設 / 全作業 / 土工 / 型枠 / 掘削 / 運転・交通 / 重機作業 / 電気作業 / 高所作業
天候・季節冬・冷え込み / 夏・猛暑 / 強風 / 花粉
対象者全員向け / 新規入場者向け / 運転手向け
雰囲気場を和ませる（笑い・クイズ）/ 学べる雑学 / 数字で語る / 気を引き締める（法令・罰則・統計）/ 演出・パフォーマンス

使い方

  1. 絞り込みたいタグを順番にタップ。選択中のタグは黄色になります。
  2. 「過去1ヶ月に使ったネタを除外」のチェックを入れると、最近読んだネタは候補から外れます。
  3. 表示件数が右上に出ます。条件を緩めたい時は黄色のタグをもう一度タップして解除。

厳密一致モード：選んだタグを全て含むネタだけ表示されます。「全作業」だけは特別ではなくなりました（以前のフォールバックは廃止）。

6ネタを開いた画面（モーダル）の使い方

どこかのカードをタップすると、画面下から本文が出てきます（モーダル）。

上部

  • A− / A+：この画面だけ文字サイズを一時調整。朝礼で読み上げ中に便利。
  • ×：閉じる。

本文中の強調表示

  • 数値 は赤色（例：20kg、3分）
  • 行動 は黄色（例：必ず2人で持つ、指差し確認）

👍ウケた／📝役に立った ボタン

本文の下に2つの評価ボタンがあります。1ネタ1端末1回。押すと黄色く反転して「✓取消」が表示されます。もう一度押すとカウントを取消できます。

※ 端末ごとの記録です。複数の端末で別々にカウントされます。
複数端末で揃えたい時は「マイデータ バックアップ・復元」で同期できます（§8参照）。

8💾 マイデータのバックアップ・復元

各端末に保存されている「お気に入り・評価・使用履歴」を JSON ファイルで保存／復元できます。別の端末でも同じデータを使いたい時、またはツールがアップデートされた後も自分の履歴を残したい時に使います。

バックアップ手順（A端末で）

  1. ⚙️ 設定タブ → 「💾 マイデータのバックアップ・復元」
  2. ⬇️ マイデータをバックアップ をタップ
  3. chorei-mydata-YYYY-MM-DD-HH-MM-SS.json という名前のファイルがダウンロードされます
  4. そのファイルを AirDrop / iCloud Drive / メール / LINE などでB端末に送る

復元手順（B端末で）

  1. 受け取った JSON ファイルを B端末に保存
  2. ⚙️ 設定 → 「💾 マイデータのバックアップ・復元」
  3. ⬆️ マイデータを復元 をタップ
  4. JSON ファイルを選択
  5. 確認ダイアログで OK＝マージ または キャンセル＝全置換 を選ぶ
    • マージ（推奨）：既存データに上書きする。両方の端末で評価したネタは合算ではなく上書き。
    • 全置換：B端末の既存データを消して、JSONの内容だけにする。

✅ 安心ポイント

ツールがアップデートされても、この機能で自分のデータは保護できます。私（運営側）がコード変更してもマイデータは消えません。

10朝礼看板に貼る（QR）

現場の朝礼看板や休憩所にQRコードを貼っておけば、職人さん全員が自分のスマホでツールを開けます。

  1. ⚙️ 設定タブ → 「朝礼看板用 QRコード」
  2. 表示されているQRを長押し → 画像保存（または右クリック保存）
  3. A4サイズくらいに印刷
  4. 看板や休憩所、ヘルメット入れの近くに掲示

💡 印刷時のコツ

「QR：現場朝礼ネタ・無料・スマホで読み取ってね」など説明文を添えると食いつきが良くなります。

  2. パスワード入力 → 次へ
  3. PIN入力 → ログイン
  4. 右上に 管理 バッジが点灯、編集機能が有効化

🔑 初期パスワード/PIN

※ ログイン後すぐ PW変更 PIN変更 でご自身用に変更してください。
※ 端末ごとのlocalStorageに保存されます。

ネタの追加

  1. 管理者ログイン後、画面右下に ＋ ボタンが現れます
  2. タップして編集画面 → タイトル・カテゴリ・本文・タグ・該当月を入力
  3. 本文には強調タグが使えます：
    <b class="num">数値</b> → 数値（赤）
    <b class="act">行動</b> → 行動（黄）
  4. 💾 保存 → 端末バッファに記録
  5. 設定 → ☁️ ネット公開 → 📤 編集内容を GitHub に反映 で公開

ネタの編集・削除

管理者モードで該当ネタを開くと、本文下に ✏️ 編集 🗑️ 削除 が出ます。

📋 ネタ一覧管理

設定 → 「📋 ネタ一覧管理」→ ネタ一覧を開く で全155件＋を一覧。検索・編集・削除が一画面で可能。

☁️ GitHub反映の事前設定

設定 → 「☁️ ネット公開（GitHub）」
GitHub PAT（Personal Access Token）を入力して 保存。これで 📤 GitHub反映 が機能します。

💾 データ管理

  • 🔄 最新データを再取得：他端末でこちらが反映した最新を取り込む
  • 📥 neta_data.js で書き出し：バックアップ用エクスポート
  • 📤 JSON読込み：バックアップからネタ一括復元
  • ⚠️ 端末の編集中バッファをクリア：未公開の編集を全て破棄

12困ったとき・よくある質問

Q. 音声読み上げが動かない

A. ブラウザの設定で音声がOFFになっているか、その端末が日本語TTS非対応の可能性。Safari / Chrome の最新版で改善することが多いです。iPhoneは「設定 → アクセシビリティ → 読み上げコンテンツ」で日本語の音声が入っているか確認してください。

Q. 「今日のおすすめ」が同じネタばかり出る

A. 一度モーダルを開いて「✅ 使った」を押すと、しばらく候補から外れます。または別のカテゴリボタンをタップしてからホームに戻ると変わります。

Q. 別の端末でも同じお気に入りを使いたい

A. §8 のバックアップ/復元機能を使ってください。アカウント機能は無料運営のため非搭載です（クラウド同期は有料化のため）。

Q. LINE Botからキーワード送ったのに返信が来ない

A. 1～2分待っても返信がなければ、もう一度送り直してください。一時的にサーバが応答不可な場合があります。それでも届かない時は本ツール内の「💬LINE公式 友だち追加」ボタンから直接運営に問い合わせください。

Q. 管理者PW・PINを忘れた

A. Chrome開発者ツール（F12）→ Application → Local Storage → chorei_pwhash と chorei_pinhash を削除してリロードすると、初期値（Chorei2026! / 2580）に戻ります。

Q. ネタを編集したのに公開されない

A. 端末バッファに記録されているだけです。「⚙️ 設定 → ☁️ ネット公開 → 📤 編集内容を GitHub に反映」を必ず押してください。「💾 未反映の変更があります」というバナーが画面上部に出るのが目印。

Q. 看板用QRが小さくて読みづらい

A. 設定画面のQR画像を長押し→画像保存して、大きく印刷してください。A4サイズ全面に拡大すれば3mくらい離れていても読めます。

Q. ツールが急に動かなくなった

A. ブラウザのキャッシュをクリア（Cmd+Shift+R / Ctrl+Shift+R）してから再アクセスしてください。それでもダメな場合は1～2時間後に再試行。GitHub Pages側の一時障害の可能性があります。
